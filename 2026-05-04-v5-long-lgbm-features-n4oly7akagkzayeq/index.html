v5_long_lgbm の特徴量と予測パイプライン (全 33 項目)

2026-05-04 会話セッションより

概要

JRA 重賞の芝長距離 (3200m など) を対象にした 3 着内予測モデル。オッズ・人気は一切使わない (odds-free) 設計で、過去成績と馬属性のみから「3 着以内に入る確率」を出す。

  • アルゴリズム: LightGBM (binary classification) + isotonic calibration + race-level normalize
  • ラベル: 1 if finish_position ∈ [1, 3] else 0
  • 特徴量数: 33 個 (numeric 27 + categorical 6)
  • 学習データ: 過去 G1/G2/G3/L/OP の芝長距離レース (170 レース弱、5000+ 行)
  • train cutoff: 2024-12-31 (2025 以降は test/forward test)

入力データ

1 サンプル = (race × horse) の組。各サンプルに以下を含める:

  • race: distance, surface, venue, trackCondition, weather, grade, raceName, 出走頭数
  • horse: number, name, jockey, trainer, age, sex, weightToCarry (斤量), bodyWeight, father (父), motherFather (母父)
  • pastResults: 直近 10 戦の (date, venue, distance, surface, position, jockey, lastThreeFurlongs, cornerPositions)
  • label: その race での finish_position (3 着以内なら 1)

※ Horse.odds / Horse.popularity / pastResults[i].popularity は export 段階で物理的に除外。学習時にも predict 時にも一切渡らない。

Tier 5(a): 騎手スコア (3 項目)

データソース: 全 race の全 horse の pastResults を flatten して (jockey, date, distance, surface, venue, position) の long table を作る (新規スクレイプ不要)。

各 (race, horse) について race_date より前の 3 年窓に絞って集計し、Bayesian smoothing で安定化:

smoothed_rate = (hits + α · prior) / (n + α)
名前意味計算詳細
jockey_top3_long_2400長距離での騎手力3y 窓 × 芝 2400m+ × 該当 jockey rides の 3 着以内率 (smoothed, α=5)
jockey_top3_course当該会場での騎手力3y 窓 × 該当 venue × 該当 jockey rides。n < 3 なら _long_2400 にフォールバック
jockey_n_long_rides長距離騎乗経験log1p(3y 窓内の 芝 2400m+ 騎乗回数) (経験量を圧縮)

prior は「同じ race_date における全騎手の平均」を使う (slice-matched prior)。default α = 5。

データソース: scraped corpus 内の Horse.trainer × race-result から集計。pastResults には trainer 情報がないため騎手と違って一段サンプルが少ない (jockey の 1/3〜1/5)。

「ステイヤー特化系統」(例: ステゴ系) の signal は long_2400 と overall の 対比で取れる (long が overall より高ければ長距離向き)。窓は jockey/trainer (3y) より長く 5y、世代横断的な signal を取るため。

予測パイプライン

学習済みモデル + キャリブレータを使って 1 レースの予測を作る流れ:

raw 特徴量 → LightGBM → p_raw
   ↓ isotonic
p_cal (calibrated probability)
   ↓ race-level normalize: p_cal × (3 / Σp_cal)
p_norm (Σ p_norm = 3 ← 3 着分の確率質量)
   ↓ 全 nC2 ペア生成 + R 補正
jointP = p_norm_a × p_norm_b × R   (Σ jointP = 3)
   ↓ wide odds と組み合わせ
EV = jointP × wideOdds

step ごとの意味

  1. p_raw (LightGBM 生確率): 木のリーフを足し算したスコアを sigmoid したもの。3 着内入りの確率の 近似だが、絶対値はキャリブレーションされていない。
  2. p_cal (isotonic キャリブレーション): train の OOF (cross-validation の hold-out) で p_raw → 実際の 3 着内率 の単調変換を学習し、予測時にこれを適用。例えば「p_raw=0.4 のサンプル群が実は 0.55 で 3 着内に入っている」なら、p_raw=0.4 を 0.55 にズラす。生 LGBM の絶対値の歪みを単調関数で補正する古典的手法。
  3. p_norm (race normalize): 各レース内で p_cal × (3 / Σp_cal) をかけて合計を 3 にする。「3 着以内の枠が 3 つあるので確率の合計は 3」という制約を強制。レース間の難易度差を打ち消す効果もある。
  4. jointP (ペア確率): 2 頭が 同時に 3 着以内に入る確率の推定値。素朴に p_norm_a × p_norm_b とすると合計が 3 にならないので、R = 3 / Σ(p_a × p_b) で再正規化する (Σ jointP = 3 → 全ペアの 3 着内同時発生の合計 = 3)。
  5. EV (期待値): jointP × wideOdds。ワイド配当 (両方 3 着内なら払戻し) との掛け算で、買えば理論上どれだけ戻るかの期待値。S2 戦略では使わない (確率で並べる)。S1 戦略はこれの降順。

設計上の特徴 (なぜこの作りか)

  • odds-free を貫く: 「市場価格に引っ張られず純粋なスキル評価を出す」が目的。S2 戦略 (jointP 順) と組み合わせて初めて意味を持つ (前回の検証では「乖離順 = EV 順」は test 期間で全敗)。
  • 3 階層 (raw/cal/norm) の確率: モデル出力をそのまま使わず、calibration → normalize の 2 段階で「現実の頻度」と「制約」に合わせる。これが pair の jointP を解釈可能 (Σ=3) にする鍵。
  • Bayesian smoothing をあちこちで使う: 騎手・厩舎・系統のいずれもサンプルが少ないと過学習しやすいので、prior にシュリンクする (n=0 なら prior 値、n→∞ なら経験率)。α は信頼度 (擬似観測数)。
  • slice-matched prior: 「全馬の平均」ではなく「同じ race_date 時点での平均」を prior にする。time-leak を防ぐため。
  • fallback hierarchy: course 別 (n<3) → long_2400 にフォールバック。ローカル signal が薄ければグローバルに退く。
  • 過去 N 戦の rolling window: 直近の調子を反映。N=5 がデフォルトだが --n-recent で可変。pastResults は最大 10 戦保存しているので 10 まで広げられる。
  • 4 角通過順を採用: 脚質 (逃げ/差し) の代理変数。"5-5-3-2" という文字列の最後を取る。
  • 系統 lookup は手動 table: 父名から 8 系統に bucketize する lineage.py がある。父名そのものを categorical にすると cardinality が爆発するので bucket 化必須。

限界 / 弱点

  • サンプルが小さい: 170 レース弱、3 着内 positive 211 件。 statistical power が低く、test 期間 19 レース → bootstrap CI 下限が 0.3〜0.5 と弱い。
  • 「3 着内」binary: 1-2-3 着の重み付けがない。複勝・ワイドに最適化されているが単勝には弱い (順位を直接推定していない)。
  • 馬場バイアスを取り込んでいない: DailyTrackBias テーブルは別にあるが、特徴量には未統合。
  • レースのペース・展開を直接表現していない: 過去の上がり 3F 平均はあるが、「前残りレース」「ハイペース」のような race-level コンテキストは categorical にしか入っていない。
  • raceClass の grade 情報が pastResults にない: 当初予定していた jockey_top3_graded / lineage_top3_graded は実装不可で、distance ベースの _long_2400 で代替。
