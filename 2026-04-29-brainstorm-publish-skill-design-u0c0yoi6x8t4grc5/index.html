サマリ

  • 議論の目的: 各プロジェクトでの壁打ち内容をワンコマンドで HTML 化し、URL を知っている人だけ閲覧できる形で共有できるスキル brainstorm-publish の設計。
  • 主な決定事項:
    • ホスティング: GitHub Pages（公開リポジトリ）+ 推測困難なランダムパス + <meta name="robots" content="noindex,nofollow">
    • リポジトリ: himaji/brainstorm-archive（1リポに集約、しばらくはこの運用）
    • URL 構造: https://himaji.github.io/brainstorm-archive/{YYYY-MM-DD}-{slug}-{16文字hash}/。ハッシュは長め（事実上アクセス困難）
    • 入力: 両対応。会話セッションの自然言語範囲指定（「直近1往復」「ここまでの○○の議論」）と、既存 Markdown ファイルパス指定
    • 整形: 会話モードは D-A 形式（冒頭サマリ + 会話ログ忠実版）。ファイルモードは Markdown を素直に HTML 化
    • メタデータ: タイトル/slug は自動推論、--title --slug で上書き可
    • 一覧: 一覧ページは作らない。ローカルの ~/.brainstorm-archive-index.json で自分用に管理
    • 初期化: gh CLI で himaji/brainstorm-archive を作成、Pages 有効化、ダミー index.html と .nojekyll と tools/template.html を配置
  • 未解決事項:
    • Pages のビルド完了待ち（push 後 1〜2 分）。スキル側はポーリングせず即 URL 返却 + 注記する方針で進めた
    • Markdown → HTML 変換は WSL 環境に pandoc も python3-venv も無かったため、Claude が直接 HTML を生成 する方針に切り替えた（依存ゼロ化）
