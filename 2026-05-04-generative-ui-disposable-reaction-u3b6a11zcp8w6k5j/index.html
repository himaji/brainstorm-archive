  • 議論の目的: ZDNet Japan の 「従来のユーザーインターフェースが消えていく--『使い捨て』UIの到来」（Joe McKendrick／WorkOS Michael Grinich の TypeScript AI Demo Day 2026 講演を引用）を題材に、AI 生成 UI の行く末を擦り合わせる。
  • 主な決定事項（共通認識として確認できたもの）:
    • 「製品＝能力＋モデル＋データ、UI は投影レイヤ」という再定義は概ね正しく、設計の重心は 画面 から データモデル＋ツール（API/MCP）カタログ＋コンポーネント語彙 に上がる。
    • カスタムコンポーネントとデザインシステムは「残る」どころか主戦場になる。LLM 可読な意図を持つコンポーネント集合（slot/props がデータ形に対応する）の設計が一級の仕事になる。
    • ただし「固定 UI が消える」は過剰評価。固定シェル＋動的に組み立てる領域のハイブリッドが現実解。
    • レイテンシ・決定性・コストの観点から、編集系・共同編集系・学習可能性が価値の中心の道具では、リクエスト毎再生成は乗らない。
    • 「データモデルからUIを生成」は理屈は通るが、エンタープライズの実データモデルの整備度はそれに耐えない。スキーマを first-class product として扱う仕事の難易度が想定の3倍ある。
    • API 派 vs computer-use 派は二極化したまま並存する。high-value 動詞は MCP 化、ロングテールは画面駆動という分岐が10年単位で続く。
    • 金融・医療・政府系では監査ログとアクセシビリティ認証が画面に紐づくため、規制が UI 再生成の床を作る領域がある。

記事ざっと読みました。Grinich（WorkOS）の4点と Salesforce Headless 360 の話ですね。方向性は同意ですが、いくつか強く留保つけたい点があります。

同意する側：

  • 「製品＝能力+モデル+データ、UI は投影レイヤ」という再定義は正しい。MCP / Apps SDK / generative UI の流れと整合する
  • 設計の重心が「画面」から データモデル＋ツール（API/MCP）カタログ＋コンポーネント語彙 に移る、というのは確実に来ている
  • ユーザーの言う「カスタムコンポーネントとデザインシステムは残る」は残るどころか そこが主戦場になる。LLM に無制約に HTML を吐かせるとブランドも一貫性も崩れるので、LLM 可読な意図を持ったコンポーネント集合（slot/props がデータ形に対応する）を整える仕事は増える

そこは言い過ぎでは、と思う側：

  1. 「固定UIが消える」は過剰。一回限りの問い合わせや遷移型ワークフローには disposable UI が刺さるが、Figma/Notion/スプレッドシートのような 持続する成果物・共同編集・学習可能性 が価値の中心の道具では、毎回再生成は逆効果。1年かけて覚えたショートカットが揮発する道具は使われない
  2. レイテンシと決定性のコスト。5秒で答えが出ればよい質問には合うが、200ms フィードバックの編集体験には乗らない。現実解は「固定シェル + 動的に組み立てる領域」のハイブリッドで、AI SDK の generative UI も実際そうなっている
  3. スキーマがダメだとこの話は全部成立しない。「ドキュメント化されたデータモデルからUI生成」はユーザーの指摘通りだが、ここのハードルは語られているより遥かに高い。エンタープライズの実データモデルは大抵 first-class product として整備されてない。スキーマ整備が一級の設計タスクに昇格するという話で、これは「UI設計の消失」ではなく 設計対象の上流移動 に近い
  4. API派 vs computer-use派の二極化。Grinich は「エージェントは API を好む」と言うけど、computer-use 系は逆方向に振っていて、長尾のソフトに型のついた API は結局来ない、という前提で画面操作する。high-value 動詞は MCP 化、ロングテールは画面駆動 という分岐が10年は続く
  5. 規制／監査がUI再生成の床を作る。金融・医療・政府系は監査ログとアクセシビリティ認証が特定の画面に紐づいているので、リクエスト毎再生成は構造上できない領域がある

なので路線は同意で、「UI が消える」より 設計の重心がスキーマ/ツール/コンポーネント語彙へ上がり、固定UIと生成UIのハイブリッドが定着する という解像度のほうが扱いやすいと思います。ユーザーの読みはほぼそのラインなので、追加の論点は データモデル整備が想定の3倍ハード という現場感くらい。
