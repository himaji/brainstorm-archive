1. やったこと

ワイド馬券で「オッズとモデル予測の乖離 (overlay)」を狙う戦略を 2 つ作った:

  1. multifeature — 中穴帯 (4-9 番人気) の馬の P(3着内) を 9 特徴量 LR で予測。本命 (1-2 番人気) とペアで EV ≥ 1.10 を買う
  2. allpairs — 全 16 頭の P(3着内) を予測 → 全 16C2=120 ペアの EV を計算 → topK=3 evMin=1.30 を買う (中穴×中穴含む)

両方とも素朴な人気帯ベース戦略 (B1) では負けるので、複合特徴量で「歪み」を見つけにいく方針。

2. 学習した特徴量重み (正規化空間)

LR は線形モデルなので各特徴量の重み w が「sigmoid 入力への影響度」を直接表す。Z-score 正規化済みなので、|w| の絶対値で feature 間比較できる。bias は base log-odds (≒ 全体 hit 率)。

multifeature モデル (中穴帯のみ学習, 5+ 倍)

bias: -1.4278  (sigmoid → 0.193 ≒ 中穴帯馬の base 3着内率)
  logOdds         w=-0.2286   ← オッズ高い = 弱い
  popularity      w=-0.2377   ← 人気下 = 弱い (logOdds とほぼ同じ意味)
  distFit         w=-0.0043   ← 距離適性 (ほぼゼロ)
  courseFit       w=+0.0170   ← コース適性 (微弱)
  gradeExp        w=+0.0774   ← 重賞経験 (中)
  recentForm      w=-0.0327   ← 前走着順下 = 弱い
  age             w=+0.0046   ← 馬齢 (ほぼゼロ)
  lastDistance    w=+0.1093   ← 前走距離長 = 強い (★)
  last3fAvg       w=-0.0260   ← 上り 3F 速い = 強い

3. 特徴量別の解説

各特徴量について、何を測っているか / 学習結果はどう / なぜそうなった / 実用的な意味 を整理。

logOdds (単勝オッズの log)

  • 何: 1.5 倍 → 0.405、10 倍 → 2.30、100 倍 → 4.61. log を取るとオッズの倍率が線形に効く
  • 重み: multifeature -0.23 / allpairs -0.38 (最大級)
  • 解説: 「オッズが高い (= 人気薄) ほど 3着内に入りにくい」を学習。allpairs では中穴〜大穴も含むので signal が強くなる。中穴帯限定の multifeature では popularity との重複が分散される
  • 実用: 市場の評価をモデルに直接入れている。market efficient な部分。 overlay は logOdds が示す予測より他の特徴量が高評価な馬で出る

popularity (単勝人気ランク)

  • 何: 1, 2, 3, ... の整数. logOdds と高相関だが完全に同一ではない (16 頭立てと 8 頭立てで人気 4 番の意味が違う)
  • 重み: multifeature -0.24 / allpairs -0.36
  • 解説: logOdds とほぼ同じ方向だが、整数なので「離散的な人気」を捕捉。両方入れることで「人気の割にオッズが高い (= 大穴狙いの少額投票)」のような微妙なケースを別シグナルとして学習可能
  • 実用: logOdds と一緒に使うことで market info を multi-resolution で取り込む

distFit (距離適性 = 過去 5 走中、芝 2400m+ で 3 着内回数)

  • 何: 0-5 の整数. 過去 5 走全部 2400m+ 重賞で 3 着以内なら 5
  • 重み: multifeature -0.00 (ほぼゼロ) / allpairs +0.10
  • 解説: multifeature では効かないが allpairs では効く. これは中穴帯限定だと「中穴で距離適性ある馬」のサンプルが少ない (中穴帯馬は距離適性低い傾向). allpairs では本命や対抗にも距離適性ある馬が含まれて signal が立つ
  • 実用: 長距離適性は「市場が見落としやすい情報」ではない (人気にすでに反映されている)。それでも allpairs では w=+0.10 と中程度の正寄与
  • 注意: pastResults の cornerPositions が空のケースが多く、過去走全件で正しい馬かは要確認 (実装は date / distance / position の 3 つだけ見ている)

courseFit (コース適性 = 過去 5 走 京都/中山/阪神 3 着内回数)

  • 何: 0-5 の整数
  • 重み: multifeature +0.02 / allpairs +0.10
  • 解説: 京都・中山・阪神 (関西〜中央エリア) の長距離コース適性。allpairs で中程度の正寄与. これも中穴限定だと signal 弱い
  • 実用: 関東主場の東京・札幌は除外. 関西長距離血統馬を捕捉する効果

gradeExp (重賞経験 = 過去 5 走 中央 2000m+ 出走回数)

  • 何: 0-5 の整数. 重賞かどうかではなく「中央場所 (東京・中山・京都・阪神・中京・新潟・福島・札幌・函館・小倉) の 2000m+ 出走」を proxy として使う
  • 重み: multifeature +0.08 / allpairs -0.01 (ほぼゼロ)
  • 解説: multifeature では正だが allpairs では消える. これは中穴帯限定だと「重賞経験ある中穴 = 力がある」が signal になるが、 全馬対象だと既に logOdds/popularity に吸収される
  • 実用: 中穴帯絞り込み戦略では「重賞経験馬に賭ける」が有効. allpairs 戦略では市場が織り込み済み

recentForm (前走着順)

  • 何: 1-15+ の整数. 欠損は 99
  • 重み: multifeature -0.03 / allpairs -0.05
  • 解説: 「前走着順低い (= 1, 2 着) → 3 着内に入る」を学習. ただし係数小さい. たぶん前走着順は popularity に既に強く反映されているので追加 signal が薄い
  • 実用: 「ローテ」「叩き」など前走の文脈情報がないと、前走着順だけでは強く効かない

age (馬齢)

  • 何: 整数. ただし Horse.age が 0 のケース多い (DB の age 列が未入力)
  • 重み: multifeature 0.00 / allpairs -0.01 (両方ほぼゼロ)
  • 解説: データ品質の問題で signal にならず. 天皇賞春は 4 歳〜古馬戦で世代差が出るが、age が DB に入ってないので使えていない
  • 実用: スクレイパー側で age を埋めるべき (TODO). 別途 「年齢」を pastResults の date 差分から推定する代替もある

lastDistance (前走距離)

  • 何: 整数 (m). 例: 2400, 3000, 3600
  • 重み: multifeature +0.11 (最大) / allpairs +0.01
  • 解説: multifeature では強い signal. 「前走で長距離を走ってきた馬」が中穴帯で 3 着内に来やすい (= ステップレース効果). allpairs では popularity に吸収されて消える
  • 実用: 中穴帯絞り込み戦略でこそ「前走長距離馬」を拾うのが効く. 天皇賞春の前哨戦 (阪神大賞典 3000m / 日経賞 2500m) を経由してきた中穴に賭けるのと整合する

last3fAvg (過去 3 走の上り 3F 平均, 秒)

  • 何: 35.0〜38.0 程度の小数. 速いほど良い (差し脚)
  • 重み: multifeature -0.03 / allpairs -0.05
  • 解説: 「上り 3F 速い → 3 着内」を弱く捕捉. 本来もっと強い signal のはずだが、3F は馬場・展開・コース・着順位置 (前 vs 後) に依存する複合値で、生値だけだと識別力が落ちる
  • 実用: 「コース別の上り 3F の偏差」「逃げ馬と差し馬の補正」をかければもっと効きそう。今は粗い

4. multifeature と allpairs での重みの違い

両モデルで signal の効き方が変わる点は重要な発見:

feature multifeature 重み allpairs 重み 解釈
logOdds + popularity -0.46 (合計) -0.74 全馬対象だと市場 info の signal が強くなる
distFit + courseFit +0.01 +0.21 距離・コース適性は本命〜大穴の判別に効く
gradeExp +0.08 -0.01 中穴帯絞り込みだけで効く (本命は重賞経験前提)
lastDistance +0.11 +0.01 中穴帯のステップレース効果
recentForm + last3fAvg -0.06 -0.10 コンディション系は弱め

示唆: - 中穴帯絞り込み (multifeature) では「重賞経験 + 前走距離長」が overlay 抽出のメインシグナル - 全ペア (allpairs) では「距離・コース適性」が市場との乖離を作る - 両戦略は本質的に異なるシグナルを使っている → 合算する意義あり

5. 戦略ごとに何を狙っているか

multifeature の overlay 構造

「中穴で能力ある馬」 = (logOdds 高 = 人気薄) かつ (gradeExp 多 + lastDistance 長) → 市場が「人気薄」として割り引きすぎている馬を抽出. 本命とペアにすると「本命が来る前提で、もう 1 頭が複勝圏に滑り込む」ストーリー。

allpairs の overlay 構造

「全馬の中で predicted P / market P が乖離している」ペアを抽出. 結果として: - 本命同士は market efficient で EV 立たない - 本命 × 中穴は EV まあまあ - 中穴同士で distFit / courseFit が高い両馬だと jointP が膨らんで EV 大. ただし配当 50-100 倍級なので hit 率は低い (1 発で ROI が動く)

天皇賞春の例では allpairs の上位 picks が全部「中穴 × 中穴」(アクアヴァーナル × ミステリーウェイ など). これは京都長距離の「3 強構成 (1.6 / 3.4 / 5.8 倍) + 4 番人気以下が 30 倍超」という極端な分布で生じた.

6. 限界

データ量

  • 70 G レース (2021-2025) / 1090 horse-race / 211 positive (3 着内ラベル)
  • Test (2025 単年) 233 horse-race / 45 positive
  • 9 特徴量 × 211 positive で LR 学習はギリギリ. LightGBM など複雑モデルは過学習濃厚

統計的有意性

  • 両戦略の Test 95%CI 下限は 0.0-0.4 程度
  • 「ROI > 1.0」は確認できるが「真の ROI が 1.3 以上」を高い確信度で言えない
  • 両期間 ROI ≥ 1.3 のセルは sweep で未発見

特徴量の死角

  • 馬場差 (dailyTrackBias テーブルあるが未利用)
  • 騎手評価 (G1 勝率、長距離成績)
  • 厩舎 (重賞勝率)
  • 父系適性 (ステイゴールド系・ハービンジャー系の長距離特性)
  • 休養週数 (前走から間隔)
  • 斤量・性別
  • 脚質 (cornerPositions から差し/逃げ判定)

モデルの死角

  • 線形モデル限定 (LR). 特徴量の interaction (例: 距離適性 × 騎手) が学習できない
  • Calibration 未調整. predicted P が真の P からズレている可能性. Platt scaling 等で補正余地
  • ペアレベル学習未試行. 現状は horse-level model + 補正係数で pair P を作っている. ペア単位で学習した方が精度上がる可能性

7. 次にやりたいこと

Phase 1: 特徴量強化

  1. 騎手スコア: ルメール・川田・武豊・戸崎・坂井 の G1 / 長距離成績
  2. 厩舎スコア: 重賞勝率
  3. 馬場差: dailyTrackBias を入れる
  4. 脚質判定: cornerPositions 末尾の通過順位を「先行/差し/追込」3 値カテゴリに

Phase 2: モデル改善

  1. LightGBM を Python 側に. interaction を学習可能に
  2. Pair-level model: 1 sample = 1 ワイドペア で binary classification
  3. Calibration: train で predicted P / true P の散布を見て補正

Phase 3: 評価強化

  1. Walk-forward CV (各レース直前まで学習 → 予測)
  2. データ拡張: 中央芝 2200m+ や全 G1 へ拡張
  3. 斤量・性別・血統 などの特徴量を試して LR で w が立つか確認

Phase 4: 戦略バリエーション

  1. 馬連 / 三連複 / 三連単展開
  2. Kelly 系のサイズ調整 (高 EV ペアに多めに賭ける)
  3. dynamic topK (レースごとに荒れ予報で K を変える)

8. 結論

  • B2 multifeature + allpairs はバックテスト両期間プラスだが統計的有意は弱い
  • 特徴量の効き方は中穴限定 vs 全馬で大きく違う (gradeExp lastDistance vs distFit courseFit)
  • 両戦略は異なる overlay を捕捉しており合算する意義あり
  • データ量・特徴量・モデル複雑度 すべてに伸びしろ。次フェーズで掘り下げる
